# 동아대학교 승학캠퍼스 교내 길찾기

## 1. 프로그램 소개

`navDay03.py`는 동아대학교 승학캠퍼스의 건물, 출입구, 주차장, 교차점 데이터를 이용해 교내 경로를 탐색하는 로컬 웹 프로그램이다.

사용자가 출발지, 경유지, 도착지를 입력하면 `navDay03.sql`을 그래프로 변환하고 다익스트라 알고리즘으로 경로를 계산한다. 결과는 카카오 지도와 경로 목록에 표시된다.

별도의 웹 프레임워크나 MySQL 연결 없이 Python 표준 라이브러리만 사용하며 기본 주소는 다음과 같다.

```text
http://127.0.0.1:8000
```

## 2. 주요 기능

- 대표 건물, 정문, 버스 정류장, 주차장 검색
- 입력값과 관계없이 전체 장소 목록을 펼쳐 선택하거나 장소 코드를 직접 입력
- 출발지와 도착지 맞바꾸기
- 여러 경유지 추가 및 삭제
- 브라우저 GPS를 이용한 현위치 선택
- 거리 우선 또는 시간 우선 탐색
- 차량 또는 도보 이동 방식 선택
- 계단 최소화 및 실내 위주 옵션
- 엘리베이터가 연결된 경로를 우선하는 옵션
- 차량 도로의 단방향과 양방향 구분
- 차량 이동 후 주차장에서 도보로 전환
- 구간별 거리, 시간, 계단, 엘리베이터, 실내 여부 표시
- 카카오 지도 일반 지도와 위성 지도 전환
- 차량 경로와 도보 경로의 색상 구분
- 경로 순서를 숫자 마커로 표시
- 숫자 마커에 엘리베이터 `↕` 아이콘 표시
- 검색 영역과 경로 목록 접기 및 펼치기
- 반복 조작으로 실행되는 숨은 UI 효과

## 3. 파일 구성

| 파일 | 역할 |
| --- | --- |
| `navDay03.py` | SQL 로딩, 경로 탐색, HTTP API, 웹 UI를 담당하는 실행 파일 |
| `navDay03.sql` | 장소, 도로, 도형 웨이포인트 초기 데이터 |
| `navDay03.md` | 프로그램과 데이터 구조를 설명하는 문서 |
| `.env` | 로컬 카카오 JavaScript 키를 저장하는 Git 제외 파일 |

프로그램은 MySQL에 직접 접속하지 않는다. 시작할 때 `navDay03.sql`의 `INSERT`와 `UPDATE` 문을 읽어 Python 객체로 변환한 뒤 메모리에서 사용한다.

따라서 SQL을 수정한 뒤에는 실행 중인 서버를 다시 시작해야 변경 내용이 반영된다.

| 구분 | 개수 |

| 도로 | 171개 |

| 도보 도로 | 123개 |

모든 `road_data`의 시작점과 끝점은 현재 장소 데이터에 등록되어 있으며, `place_data`에는 좌표가 없는 장소가 없다.

## 5. place_data

`place_data`는 검색, 지도 마커, 경로 계산에 사용하는 장소를 저장한다.

| 컬럼 | 자료형 | 설명 |
| --- | --- | --- |
| `place_id` | VARCHAR | 장소 코드이자 기본 키 |
| `place_name` | VARCHAR | 화면에 표시할 장소 이름 |
| `latitude` | DOUBLE | 위도 |
| `longitude` | DOUBLE | 경도 |
| `altitude` | DOUBLE | 고도, 값이 없으면 `NULL` |

### 장소 코드 규칙

| 코드 | 의미 | 예시 |
| --- | --- | --- |
| `GATE` | 캠퍼스 정문 | `GATE` |
| `Sxx` | 대표 건물 | `S06` |
| `Sxx-...` | 건물 출입구 또는 층별 연결 지점 | `S06-06F03E` |
| `BSxx` | 버스 정류장 | `BS06` |
| `PLxx` | 주차장 | `PL07`, `PL06-01` |
| `CRxx` | 교차점 또는 도로 연결점 | `CR07-01` |
| `ELxx` | 엘리베이터 | `EL06` |
| `STxx` | 계단 연결점 | `ST03` |

검색 가능한 대표 장소는 다음과 같다.

```text
GATE
S01 ~ S22, S31
BS01, BS04, BS06, BS11, BS15, BS31
PL01, PL03, PL04, PL06-01, PL06-02, PL06-03, PL07, PL19
```

출발지와 도착지 입력창을 열면 현재 입력값과 관계없이 현위치를 포함한 전체 선택 목록을 표시한다. 사용자는 목록을 스크롤해 선택하거나 장소 코드를 직접 입력할 수 있다.

기본 지도에는 대표 건물, 정류장, 주차장만 표시하고, 출입구와 교차점은 경로 검색 결과에 포함될 때 표시한다.

## 6. waypoint_data

`waypoint_data`는 도로의 굴곡을 표현하기 위한 중간 좌표를 저장하는 테이블이다. 사용자가 입력하는 경유지와 이름이 비슷하지만 역할이 다르다.

| 구분 | 프로그램 내부 의미 |
| --- | --- |
| `GEOMETRY_WAYPOINTS` | 도로 모양을 표현하기 위한 숨은 중간 지점 |
| 사용자 경유지 | 출발지와 도착지 사이에서 반드시 방문할 대표 장소 |

| 컬럼 | 자료형 | 설명 |
| --- | --- | --- |
| `place_id` | VARCHAR | `WP1`과 같은 웨이포인트 코드 |
| `place_name` | VARCHAR | 웨이포인트 이름 |
| `latitude` | DOUBLE | 위도 |
| `longitude` | DOUBLE | 경도 |

현재 `WP1`부터 `WP112`까지 코드가 준비되어 있지만 좌표는 모두 `NULL`이다. 따라서 현재 경로 선의 굴곡에는 사용되지 않는다.

## 7. road_data

`road_data`의 각 행은 두 장소 사이의 이동 가능한 구간을 나타낸다.

| 컬럼 | 자료형 | 설명 |
| --- | --- | --- |
| `road_id` | INT | 도로 고유 번호 |
| `from_place` | VARCHAR | 시작 장소 코드 |
| `to_place` | VARCHAR | 끝 장소 코드 |
| `road_type` | VARCHAR | `차량` 또는 `도보` |
| `distance` | INT | 거리, 단위는 m |
| `time` | INT | 이동 시간, 단위는 초 |
| `twoway` | BOOLEAN | `1`이면 양방향, `0`이면 정방향만 허용 |
| `indoor` | BOOLEAN | `1`이면 실내 구간 |
| `stair` | BOOLEAN | `1`이면 계단 구간 |
| `slope` | INT | 경사 비용, 값이 없으면 `NULL` |
| `curve` | INT | 곡선 표현 값, 값이 없으면 `NULL` |

`twoway=0`이면 `from_place → to_place` 방향으로만 이동할 수 있다. `twoway=1`일 때만 반대 방향 간선을 추가한다.

현재 `curve` 값은 모두 `NULL`이므로 지도에는 각 장소 좌표를 직선으로 연결한다.

## 8. 대표 건물과 출입구

사용자는 `S06`과 같은 대표 건물을 선택하지만 실제 경로는 도로에 연결된 출입구를 사용한다.

예를 들어 `S06`을 선택하면 프로그램은 `S06-`으로 시작하면서 도로에 연결된 장소와 같은 건물 번호의 `EL06`을 출입구 후보로 수집한다.

```text
S06-04F
S06-06F01E
S06-06F02E
S06-06F03E
EL06
```

여러 후보 중 거리 또는 시간 비용이 가장 낮은 출입구를 자동으로 선택한다. 경로 목록의 출발지와 도착지에는 내부 출입구가 아니라 사용자가 선택한 대표 건물 정보를 표시한다.

대표 건물과 출입구 사이에 별도의 가상 도로를 만드는 방식은 아니다. 대표 건물을 해당 접두사의 출입구 후보 묶음으로 처리한다.

## 9. 경로 탐색

프로그램은 장소를 정점, 도로를 간선으로 구성하고 다익스트라 알고리즘으로 최단 경로를 계산한다.

### 도보 모드

1. `road_type`이 `도보`, `보행`, `walk`, `도로`인 구간만 사용한다.
2. 거리 또는 시간을 탐색 비용으로 사용한다.
3. 계단 최소화가 켜지면 계단 구간에 큰 추가 비용을 적용한다.
4. 실내 위주가 켜지면 실외 도보 구간에 추가 비용을 적용한다.
5. `twoway`에 따라 이동 방향을 제한한다.

### 차량 모드

차량 모드는 필요한 경우 도보와 차량을 순서대로 결합한다.

```text
건물 출입구 → 주차장        도보
주차장 → 목적지 인근 주차장 차량
주차장 → 목적지 출입구      도보
```

- 출발점이 차량 도로에 직접 연결되면 바로 차량 탐색을 시작한다.
- 출발점이 건물 출입구이면 도보로 주차장까지 이동한 뒤 차량으로 전환한다.
- 차량 이동 후 `PL...` 주차장에서만 도보로 전환한다.
- 같은 주차장에서 차량을 시작하자마자 다시 도보로 전환하는 경로는 허용하지 않는다.
- 차량 구간이 목적지 후보에 직접 도착하면 도보 전환 없이 종료할 수 있다.

### 경유지

경유지가 있으면 다음처럼 구간을 나눠 각각 탐색한 뒤 결과를 합친다.

```text
출발지 → 경유지 1 → 경유지 2 → 도착지
```

경로 목록에는 경유지를 실제 순서에 맞춰 구간 사이에 표시한다.

### 엘리베이터 우선

엘리베이터 우선 옵션을 켜면 `EL...` 지점에 들어간 뒤 다른 출입구로 나오는 경로를 먼저 찾는다.

- 도로에 연결된 모든 `ELxx`를 동일한 우선 후보로 사용한다.
- `ELxx`는 같은 번호의 `Sxx` 건물 출입구 후보에도 포함한다.
- 일반 최단 경로 비용의 3배 이내에서 이용 가능한 엘리베이터 수가 많은 경로를 우선한다.
- 한 경로에서 우선 이용하는 엘리베이터는 최대 4개이며, 이용 개수가 같으면 거리·시간 비용이 낮은 경로를 선택한다.
- EL 지점에 들어간 뒤 다른 연결점으로 나와야 실제 엘리베이터 이용으로 계산한다. 같은 출입구로 되돌아오거나 같은 EL을 다시 이용하는 경로는 제외한다.
- 목적지 건물의 EL에 도착하면 해당 지점을 출입구로 인정하고 경로를 종료할 수 있다.
- 목적지 후보에 도착한 뒤에는 목적지 건물의 EL 연결만 추가 탐색하고 다른 건물의 EL을 찾으러 나가지 않는다.
- EL과 직접 연결된 도로는 SQL의 `stair=1` 여부보다 엘리베이터 구간 판정을 우선한다. 계단 최소화 비용에서 제외하고 화면에는 `엘리베이터`로 표시한다.
- 대표 건물에서 출발할 때는 대표 좌표와 가장 가까운 연결 출입구를 시작점으로 사용한다.
- 엘리베이터가 없는 건물의 `STxx` 경로는 그대로 사용할 수 있으며, 조건에 맞는 엘리베이터 경로가 없으면 일반 경로로 자동 복귀한다.

## 10. 현위치 처리

브라우저 Geolocation API로 받은 좌표는 SQL 장소가 아니라 `CURRENT_LOCATION`이라는 임시 장소로 처리한다.

1. 현재 좌표에서 가까운 연결 장소 최대 5개를 찾는다.
2. 각 연결 후보를 시작점 또는 도착점 후보로 사용한다.
3. 현재 좌표와 선택된 연결점 사이는 하버사인 거리로 계산한 임시 도보 구간으로 연결한다.

현위치 연결은 실제 보행로 데이터가 아니라 직선거리이므로 건물이나 담장을 가로지를 수 있다. 정확한 경로가 필요하면 현위치 주변 도보 연결 데이터를 추가해야 한다.

## 11. 경로 옵션

| 옵션 | 처리 방식 |
| --- | --- |
| 거리 우선 | `distance`를 기본 비용으로 사용 |
| 시간 우선 | `time`을 기본 비용으로 사용 |
| 계단 최소화 | 도보 계단 구간에 큰 추가 비용 적용 |
| 실내 위주 | 실외 도보 구간에 추가 비용 적용 |
| 엘리베이터 우선 | 가능한 경우 서로 다른 출입구를 EL 지점으로 연결하는 경로 선택 |
| 경사 | `slope` 값이 있으면 비용에 추가 |

SQL의 시간은 초 단위로 저장하며 화면에서는 60초 미만은 초, 60초 이상은 올림한 분으로 표시한다.

## 12. 웹 API

| 주소 | 설명 |
| --- | --- |
| `/` | 교내 길찾기 웹 화면 |
| `/api/places` | 검색 가능한 대표 장소 |
| `/api/map-places` | 전체 지도 장소 |
| `/api/roads` | 전체 도로와 방향·속성 정보 |
| `/api/route` | 경로 탐색 결과 |

`/api/route`의 파라미터는 다음과 같다.

| 파라미터 | 값 | 설명 |
| --- | --- | --- |
| `start` | 장소 코드 | 출발지 |
| `end` | 장소 코드 | 도착지 |
| `waypoints` | 쉼표로 구분한 코드 | 사용자 경유지 목록 |
| `weight` | `distance`, `time` | 탐색 기준 |
| `travel_mode` | `car`, `walk` | 이동 방식 |
| `avoid_stairs` | `0`, `1` | 계단 최소화 |
| `rainy` | `0`, `1` | 실내 위주 |
| `prefer_elevator` | `0`, `1` | 엘리베이터 우선 |
| `current_lat` | 실수 | 현위치 위도 |
| `current_lng` | 실수 | 현위치 경도 |

서버는 같은 클라이언트가 10초 동안 전체 요청 150회를 초과하면 `429 Too Many Requests`를 반환한다. 메인 화면(`/`)은 별도로 10초에 5회까지 허용하므로, 화면을 연 뒤 새로고침을 5번 연속 실행하면 다음 화면 요청이 차단된다.

모든 정상 응답에는 `Cache-Control: no-store`를 넣어 Python 파일이나 SQL 변경 후 브라우저가 이전 HTML과 API 결과를 재사용하지 않도록 한다.

## 13. 지도 표시

카카오 지도 JavaScript API를 사용하며 검색된 경로는 다음처럼 표시한다.

- 차량 경로 선과 숫자 마커: 파란색
- 도보 경로 선과 숫자 마커: 초록색
- 차량에서 도보로 바뀌면 선과 번호 색상도 함께 변경
- 경로의 각 장소를 이동 순서 번호로 표시
- 경로 번호가 EL 지점이면 오른쪽 위에 보라색 `↕` 아이콘 표시
- 같은 좌표의 번호가 겹치면 좌우 위치 조정
- 전체 경로가 보이도록 지도 범위 자동 조정
- 일반 지도와 위성 지도 전환 지원

## 14. UI 이스터에그

- 출발지·도착지 스왑 버튼을 10번 누르면 `스왑버튼을대체몇번을누르는거야으아악아악아악아악` 문장을 한 음절씩 브라우저 알림창으로 표시한다. 모든 창을 닫으면 횟수를 초기화한다.
- 위 입력 패널 접기 버튼과 왼쪽 경로 패널 접기 버튼은 각각 10번 누르면 해당 버튼에 무지개 애니메이션을 적용한다.
- 네 번째 이후 경유지 입력칸에는 `대체 얼마나 더 경유를 하실려고요?`를 선택된 기본값으로 넣어 바로 덮어쓸 수 있게 한다.

## 15. 실행 방법

### API 키 설정

`Chapter03-Day03/.env` 파일에 카카오 JavaScript 키를 저장한다.

```text
KAKAO_MAP_API_KEY=카카오_JavaScript_키
```

프로그램은 환경변수를 먼저 확인하고 값이 없으면 실행 파일과 같은 폴더의 `.env`를 읽는다. `.env`는 Git에 올리지 않는다.

### 서버 실행

프로젝트 루트에서 다음 명령을 실행한다.

```powershell
python Chapter03-Day03/navDay03.py
```

또는 폴더로 이동해서 실행한다.

```powershell
cd Chapter03-Day03
python navDay03.py
```

브라우저에서 다음 주소로 접속한다.

```text
http://127.0.0.1:8000
```

### 서버 종료

실행 중인 터미널에서 `Ctrl+C`를 누른다. VS Code 실행 중지 또는 실행 터미널 종료로 부모 프로세스가 끝나면 Windows에서 서버도 자동으로 종료된다.

8000번 포트에 기존 서버가 있으면 새 서버를 실행할 수 없다. 기존 서버를 종료한 뒤 다시 실행해야 한다.

## 16. API 키 관리

- 키를 Python, SQL, Markdown 파일에 직접 작성하지 않는다.
- `.env`와 `.env.*`를 `.gitignore`에 포함한다.
- 카카오 개발자 콘솔의 Web 플랫폼에 실제 접속 주소를 등록한다.
- `localhost`와 `127.0.0.1`은 서로 다른 도메인으로 등록해야 한다.
- 공개 저장소에 노출된 키는 재발급한다.

## 17. 현재 제한 사항

- `waypoint_data`의 112개 좌표가 모두 `NULL`이므로 실제 도로 곡선을 표현하지 못한다.
- 모든 `curve` 값이 `NULL`이라 곡률 기반 경로 표시는 아직 적용되지 않는다.
- 지도 선은 등록된 장소 좌표 사이를 직선으로 연결한다.
- 차량에서 도보로 전환할 수 있는 곳은 현재 `PL...` 주차장으로 제한된다.
- 대표 장소가 존재해도 해당 건물 출입구가 도로 그래프와 연결되지 않으면 경로를 찾지 못할 수 있다.
- 현위치와 첫 도로 연결점 사이는 실제 보행로가 아닌 직선 연결이다.
- 서버는 실행 시 SQL을 한 번만 읽으므로 데이터 수정 후 재시작해야 한다.
