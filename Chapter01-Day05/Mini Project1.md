# 동아대학교 교내 길찾기 미니 프로젝트

## 프로젝트 개요

이 폴더는 동아대학교 캠퍼스의 장소와 이동 경로를 그래프로 구성하고 최단 경로를 찾기 위해 작성한 초기 미니 프로젝트 코드이다.

캠퍼스의 건물, 버스 정류장, 계단, 주차장, 야외 공간 등을 노드로 표현하고 두 장소 사이의 도보, 엘리베이터, 실내 통로를 간선으로 연결한다. 이 데이터를 이용해 네트워크 그래프를 그림으로 확인하거나 예상 시간이 가장 짧은 경로를 계산하는 것이 프로젝트의 목적이다.

## 폴더 구성

| 파일 | 내용 |
| --- | --- |
| `donga.py` | 캠퍼스 인접 관계를 도로 데이터로 변환하고 NetworkX 그래프로 시각화한다. |
| `full_pages.py` | 장소 이름과 도보·엘리베이터·건물 연계 간선 데이터를 한곳에 정의한다. |
| `roadfind.py` | 장소 목록과 다익스트라 알고리즘을 이용해 최단 시간 경로를 탐색한다. |
| `Mini Project1.md` | 프로젝트의 구성과 각 파일의 역할을 정리한다. |

## donga.py

`donga.py`는 이미지에서 정리한 캠퍼스 장소의 인접 관계를 무방향 그래프로 만드는 코드이다. 코드 상단에는 역할이 `도메인전문가`로 기록되어 있다.

주요 처리 내용은 다음과 같다.

1. 정문, 108계단, 차고지, 체대, 농구장, 건물 등의 인접 관계를 `adjacency`에 정의한다.
2. 같은 장소 연결이 중복되지 않도록 정리한다.
3. 각 연결에 `r01`, `r02` 형식의 도로 ID를 부여한다.
4. 생성한 도로 정보를 `road_data.txt`에 저장한다.
5. NetworkX의 무방향 그래프로 노드와 간선을 구성한다.
6. Matplotlib으로 그래프를 그리고 `campus_graph.png`로 저장한다.

Windows, macOS, Linux에서 한글이 표시되도록 사용 가능한 한글 폰트를 자동으로 찾는다. Linux에서 폰트를 찾지 못하면 `fonts-nanum` 설치를 시도한다.

사용 라이브러리는 다음과 같다.

```text
networkx
matplotlib
```

## full_pages.py

`full_pages.py`는 장소 이름과 모든 이동 연결을 관리하는 통합 데이터 파일이다. 같은 데이터를 Excel이나 HTML 등 여러 출력 형식에서 사용할 수 있도록 하나의 소스로 정리한 코드이다.

### 장소 데이터

`NAMES` 딕셔너리에는 코드에서 사용하는 장소 ID와 화면에 표시할 이름이 저장되어 있다. 현재 등록된 장소는 35개이며, 간선에 등장하는 모든 장소가 `NAMES`에 등록되어 있다.

장소에는 다음과 같은 정보가 포함된다.

- 정문, 108계단, 차고지, 농구장, 삼각지, 야외정원
- S01 ~ S06, S11, S12 주요 건물
- 건물의 지하, 1층, 2층, 3층, 4층, 6층, 7층 연결 지점
- 한림도서관, 자연과학대학, 예술체육대학 건물
- 산학협력관 버스정류장

### 이동 연결 데이터

`EDGES`에는 총 51개의 간선이 저장되어 있다. 각 간선은 다음 형식을 사용한다.

```python
(도로 ID, 출발 장소, 도착 장소, 시간, 이동 종류, 실내 여부, 경사도)
```

예시는 다음과 같다.

```python
("r01", "정문", "108계단", 4, "도보", "N", 2)
```

이 데이터는 정문에서 108계단까지 도보로 4분이 걸리고, 실외 구간이며, 경사도 값이 2라는 뜻이다.

간선은 세 종류로 구성되어 있다.

- `r01` ~ `r33`: 캠퍼스의 일반 도보 경로
- `r34` ~ `r45`: EL01, EL04, EL06, EL07, EL10, EL17 엘리베이터 경로
- `r46` ~ `r51`: 건물 사이의 브릿지 또는 실내 통로

엘리베이터와 일부 건물 연계 구간은 원래 시간과 경사 데이터가 없어 기본값으로 시간 1분, 경사도 0을 사용한다.

`GATE_LATLNG`에는 향후 실제 지도에 표시할 정문 GPS 기준 좌표가 저장되어 있다.

```python
GATE_LATLNG = (35.1159, 128.9673)
```

파일을 직접 실행하면 장소 수, 간선에 등장하는 고유 노드 수, 누락된 장소, 간선 수를 출력한다.

## roadfind.py

`roadfind.py`는 사용자가 출발지와 도착지 ID를 입력하면 예상 시간이 가장 짧은 경로를 찾는 콘솔 프로그램이다.

### 장소 코드

`places` 딕셔너리에는 다음 종류의 장소가 정의되어 있다.

- `GATE`: 정문
- `SB01` ~ `SB06`: 버스 정류장
- `S01` ~ `S20`: 캠퍼스 건물
- `EL01`, `EL04`, `EL06`, `EL07`, `EL10`, `EL17`: 건물 엘리베이터

### 경로 탐색 과정

1. 모든 장소를 `campus_graph`의 노드로 생성한다.
2. `road_data`의 연결 정보를 양방향 간선으로 추가한다.
3. 다익스트라 알고리즘으로 출발지부터 각 장소까지의 최소 시간을 계산한다.
4. `previous` 정보를 역으로 따라가 최종 이동 경로를 만든다.
5. 장소별 이동 종류, 예상 시간, 실내 여부와 총 예상 시간을 출력한다.

`navigation()` 함수는 선택 가능한 장소를 출력하고 출발지와 도착지를 입력받는다. 존재하지 않거나 선택할 수 없는 장소가 입력되면 오류 메시지를 보여 준다.

### 현재 코드 상태

현재 `roadfind.py`는 길찾기 알고리즘의 초안이며 단독으로는 실행되지 않는다.

- `heapq` 모듈을 사용하지만 import 문이 없다.
- `road_data`가 파일 안에 정의되어 있지 않다.
- 장소 선택 여부를 나타내는 `place_type`이 정의되어 있지 않다.
- 실행 조건이 `if name == " main":`으로 작성되어 있어 `if __name__ == "__main__":`으로 수정해야 한다.
- `full_pages.py`의 `EDGES` 형식과 `roadfind.py`가 기대하는 `road_data` 딕셔너리 형식이 서로 다르다.

따라서 실제로 실행하려면 `full_pages.py`의 장소와 간선 데이터를 `roadfind.py`에서 사용할 수 있는 형태로 변환하고 누락된 import와 실행 조건을 보완해야 한다.

## 전체 흐름

```text
캠퍼스 장소와 연결 관계 정리
        ↓
donga.py로 도로 목록 및 그래프 이미지 생성
        ↓
full_pages.py에 장소·도보·엘리베이터·실내 통로 데이터 통합
        ↓
roadfind.py에서 다익스트라 알고리즘으로 최단 시간 경로 탐색
```
